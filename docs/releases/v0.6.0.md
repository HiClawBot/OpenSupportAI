# OpenSupportAI v0.6.0

## English

This minor release adds the first real LLM-backed answer path to the OpenSupportAI runtime.

### Added

- OpenAI-compatible grounded answer generation in the API orchestrator.
- Active non-demo LLM provider usage for questions that retrieve matching knowledge chunks.
- Grounded answer prompt assembly with the customer question and retrieved knowledge snippets.
- `ai_runs` metadata for provider, model, prompt version, token usage, confidence, retrieved chunk IDs, generation source, and fallback status.
- Regression coverage for configured LLM generation through the client conversation/message API flow.

### Preserved

- No-hit retrieval still refuses to invent an answer and suggests human handoff.
- `demo://local`, missing provider, failed model requests, and empty model responses fall back to deterministic grounded answers.
- Tool-first order/subscription demo flows still run before knowledge retrieval.
- Human handoff and Chatwoot handoff flows keep their existing behavior.

### Changed

- Bumped workspace package versions to `0.6.0`.
- Added `@opensupportai/llm` as an API service dependency.
- Updated README, API spec, release checklist, and changelog for the v0.6 release line.

## 中文

这个 minor 版本为 OpenSupportAI runtime 增加首条真实 LLM-backed answer path。

### 新增

- API orchestrator 支持 OpenAI-compatible grounded answer generation。
- 当用户问题检索到知识块，并且项目配置了 active、非 demo 的 LLM provider 时，会调用真实 provider 生成回答。
- Grounded answer prompt 会包含用户问题和检索到的知识片段。
- `ai_runs` 记录 provider、model、prompt version、token usage、confidence、retrieved chunk IDs、generation source 和 fallback status。
- 新增通过客户端 conversation/message API 触发已配置 LLM generation 的回归测试。

### 保留

- 无知识命中时仍拒绝编造答案，并建议转人工。
- `demo://local`、未配置 provider、模型请求失败、模型返回空内容时，仍回退到确定性 grounded answer。
- 订单/订阅 demo 工具仍会在知识库检索前优先执行。
- 人工转接和 Chatwoot handoff 流程保持现有行为。

### 变更

- Workspace package versions 升级到 `0.6.0`。
- API service 新增 `@opensupportai/llm` dependency。
- README、API spec、发布清单和 changelog 更新到 v0.6 版本线。
