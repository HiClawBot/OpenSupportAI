# OpenSupportAI v1.0.0

## English

This is the first stable public release of OpenSupportAI. It keeps the product scope focused on an embeddable AI support runtime, then freezes the public contracts needed for self-hosted operators and downstream integrators.

### Added

- v1.0 public contract documentation:
  - stable `/v1` REST API boundaries
  - stable JavaScript SDK methods
  - stable Widget initialization API
  - stable Chatwoot handoff adapter boundary
  - stable generic webhook and Slack inbound channel adapter boundaries
  - stable OpenAPI-style business tool executor boundary
- v1.0 upgrade guide for v0.5.x through v0.9.x deployments.
- v1.0 production deployment guide covering environment variables, reverse proxy requirements, database backup/rehearsal, security hardening, observability, and release validation.
- Security guidance for OpenAPI business tools, including host allowlists, bearer-env auth, default mutation blocking, and production review expectations.

### Changed

- `/v1` is now treated as a stable public API. Future patch/minor releases may add compatible fields/endpoints/events but should not remove or rename existing public response fields.
- README now presents v1.0 capabilities and links to public contracts, upgrade, and production deployment docs.
- API spec, data model, roadmap, release checklist, and security docs now reflect v1.0.
- Workspace package versions are bumped to `1.0.0`.

- `pnpm test`

- `pnpm smoke:channels -- --help`
- `pnpm smoke:tools -- --help`
- Real memory-mode smoke

### Known Limits

- OpenSupportAI v1.0 is not a CRM, ticketing suite, or all-channel inbox.
- Slack supports signed inbound Events API callbacks; Slack outbound replies remain future work.
- Email and Telegram remain adapter catalog stubs.
- OpenAPI tools are configured through the admin API; automatic OpenAPI spec import, model planning, encrypted per-tool secret storage, and human approval flows remain future work.
- Retrieval still uses deterministic keyword scoring with pgvector-ready schema support.

## 中文

这是 OpenSupportAI 的首个稳定公开发布版本。它继续保持产品范围聚焦：作为可嵌入 AI 客服运行时，并冻结自托管 operator 和二次开发者需要依赖的公共契约。

### 新增

- v1.0 公共契约文档：
  - 稳定 `/v1` REST API 边界
  - 稳定 JavaScript SDK 方法
  - 稳定 Widget 初始化 API
  - 稳定 Chatwoot handoff adapter 边界
  - 稳定 generic webhook 与 Slack 入站 channel adapter 边界
  - 稳定 OpenAPI-style business tool executor 边界
- 面向 v0.5.x 到 v0.9.x 部署的 v1.0 升级指南。
- v1.0 生产部署指南，覆盖环境变量、反向代理要求、数据库备份/演练、安全硬化、观测和发布验证。
- OpenAPI business tool 安全说明，包含 host allowlist、bearer-env auth、默认 mutation 阻断和生产审查要求。

- README 更新为 v1.0 能力描述，并链接公共契约、升级和生产部署文档。
- API 规范、数据模型、路线图、发布清单和安全文档同步到 v1.0。
