# OpenSupportAI v0.1.4

## English

This release turns the worker placeholder into a practical async-job foundation.

Highlights:

- Added Prisma `AsyncJob` storage and migration.
- Added repository methods to create, list, claim, complete, fail, and retry async jobs.
- Added admin jobs API: `GET/POST /v1/admin/projects/{project_id}/jobs`.
- Added a tested worker runtime that claims jobs, dispatches handlers, completes successful jobs, and schedules retry times after failures.
- Added API and worker tests for the async-job foundation.

Validation:

- `pnpm format:check`
- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`
- `pnpm exec prisma validate`
- `pnpm db:generate`
- `pnpm smoke:memory -- --help`
- `pnpm smoke:chatwoot -- --help`

## 中文

这个版本把 worker 从占位进程升级为可复用的异步任务基础设施。

主要更新：

- 新增 Prisma `AsyncJob` 存储和 migration。
- Repository 新增 async job 创建、列表、claim、完成、失败和重试方法。
- 管理端新增 jobs API：`GET/POST /v1/admin/projects/{project_id}/jobs`。
- 新增可测试的 worker runtime：claim job、分发 handler、完成成功任务，并在失败后安排 retry 时间。
- 新增 async-job 基础设施的 API 和 worker 测试。
