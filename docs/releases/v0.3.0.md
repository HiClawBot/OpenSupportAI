# OpenSupportAI v0.3.0

## English

This release adds the first safe business-tool layer for the support runtime.

Highlights:

- Added Prisma `ToolDefinition` and `ToolCall` storage with migration support.
- Added admin tools API:
  - `GET /v1/admin/projects/{project_id}/tools`
  - `POST /v1/admin/projects/{project_id}/tools`
  - `PATCH /v1/admin/projects/{project_id}/tools/{tool_id}`
  - `GET /v1/admin/projects/{project_id}/tool-calls`
- Added allowlist control through tool `status=active|disabled`.
- Added seeded demo tools:
  - `demo.order_lookup`
  - `demo.subscription_lookup`
- Added deterministic demo tool execution before RAG fallback for explicit order/subscription-status questions.
- Added tool-call logs to admin conversation detail and ops health counts.
- Updated the demo app prompt strip and tests for business tool workflows.

Validation:

- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`
- `pnpm exec prisma validate`
- `pnpm db:generate`
- `pnpm smoke:memory -- --help`
- `pnpm smoke:chatwoot -- --help`

## 中文

这个版本为客服运行时加入第一层安全业务工具能力。

主要更新：

- 新增 Prisma `ToolDefinition` 和 `ToolCall` 存储及 migration。
- 新增管理端 tools API：
  - `GET /v1/admin/projects/{project_id}/tools`
  - `POST /v1/admin/projects/{project_id}/tools`
  - `PATCH /v1/admin/projects/{project_id}/tools/{tool_id}`
  - `GET /v1/admin/projects/{project_id}/tool-calls`
- 通过 tool `status=active|disabled` 实现 allowlist 启停控制。
- 内置 demo 工具：
  - `demo.order_lookup`
  - `demo.subscription_lookup`
- 对明确的订单/订阅状态问题，orchestrator 会先执行确定性 demo tool，再进入 RAG fallback。
- 管理端会话详情和 ops health 增加 tool-call 日志与计数。
- 更新 Demo App 提示和业务工具工作流测试。
