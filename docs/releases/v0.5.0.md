# OpenSupportAI v0.5.0

## English

This release adds the first multi-channel adapter foundation.

Highlights:

- Added shared channel adapter protocol types in `@opensupportai/protocol`.
- Added `@opensupportai/adapter-channels`.
- Added a fully testable generic webhook inbound adapter.
- Added Slack, email, and Telegram as typed contract stubs with explicit capabilities and configuration keys.
- Added admin channel adapter APIs:
  - `GET /v1/admin/projects/{project_id}/channels/adapters`
  - `POST /v1/admin/projects/{project_id}/channels/adapters/{provider}/test`
- Added generic inbound webhook ingestion:
  - `POST /v1/channel-webhooks/generic?public_key=pk_demo`
- Generic webhook ingestion records webhook events, creates or reuses contacts/conversations, stores inbound end-user messages, and reuses the existing orchestrator response flow.
- Added `pnpm smoke:channels` for local memory-mode channel webhook validation.
- Added tests for adapter catalog, generic webhook normalization, external conversation reuse, stored messages, and webhook event visibility.

Validation:

- `pnpm build`

- `pnpm smoke:chatwoot -- --help`
- `pnpm smoke:channels -- --help`
- real memory-mode smoke
- real channel webhook smoke

## 中文

这个版本加入第一阶段多渠道 adapter 基础。

主要更新：

- 在 `@opensupportai/protocol` 中新增共享 channel adapter 协议类型。
- 新增 `@opensupportai/adapter-channels` 包。
- 新增可完整测试的 generic webhook 入站 adapter。
- 将 Slack、Email、Telegram 加入为类型化契约 stub，明确 capabilities 和 configuration keys。
- 新增管理端 channel adapter API：
  - `GET /v1/admin/projects/{project_id}/channels/adapters`
  - `POST /v1/admin/projects/{project_id}/channels/adapters/{provider}/test`
- 新增 generic 入站 webhook：
  - `POST /v1/channel-webhooks/generic?public_key=pk_demo`
- Generic webhook 入站会记录 webhook event、创建或复用 contact/conversation、写入 end-user message，并复用现有 orchestrator 回复流程。
- 新增 `pnpm smoke:channels`，用于本地内存模式 channel webhook smoke test。
- 新增 adapter catalog、generic webhook normalize、外部会话复用、消息落库和 webhook event 可见性测试。
