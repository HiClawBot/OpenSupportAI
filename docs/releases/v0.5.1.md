# OpenSupportAI v0.5.1

## English

This release hardens the v0.5 generic webhook channel foundation.

Highlights:

- Added admin configuration for generic webhook secrets:
  - `GET /v1/admin/projects/{project_id}/channels/generic-webhook`
  - `POST /v1/admin/projects/{project_id}/channels/generic-webhook`
- Configured generic webhook channels now verify the configured secret header, `X-OpenSupportAI-Webhook-Secret`, `X-Webhook-Secret`, or `Authorization: Bearer <secret>`.
- Made generic webhook idempotency project-scoped with `project_id + provider + external_event_id`.
- Re-delivered processed webhook events now return an idempotent response and do not create duplicate end-user messages.
- Added `channel` metadata to admin conversation list and detail responses.
- Expanded channel smoke testing for invalid public keys, invalid secrets, invalid payloads, idempotent duplicates, and admin channel visibility.

- real memory-mode smoke
- real channel webhook smoke

## 中文

这个版本强化 v0.5 的 generic webhook channel 基础。

主要更新：

- 管理端新增 generic webhook secret 配置：
  - `GET /v1/admin/projects/{project_id}/channels/generic-webhook`
  - `POST /v1/admin/projects/{project_id}/channels/generic-webhook`
- 配置 generic webhook channel 后，会校验配置的 secret header、`X-OpenSupportAI-Webhook-Secret`、`X-Webhook-Secret`，或 `Authorization: Bearer <secret>`。
- Generic webhook 幂等键调整为项目级 `project_id + provider + external_event_id`。
- 已处理 webhook event 重复投递时返回幂等响应，不会重复创建 end-user message。
- 管理端 conversation list/detail 响应新增 `channel` metadata。
- Channel smoke test 新增非法 public key、非法 secret、非法 payload、重复事件幂等和管理端 channel 可见性覆盖。
