# OpenSupportAI v0.9.0

## English

This minor release turns the business tool foundation into a locally verifiable OpenAPI-style HTTP executor.

### Added

- OpenAPI-style business tool execution for active project-scoped tools.
- Metadata-driven intent matching with `keywords` and regex `extract` rules.
- HTTP execution safety controls:
  - final-host allowlists through `metadata.allowed_hosts`
  - request timeouts through `metadata.timeout_ms`
  - response size limits through `metadata.max_response_bytes`
  - response object selection through `metadata.response_path`
  - rendered replies through `metadata.answer_template`
  - bearer auth through `metadata.auth = { "type": "bearer_env", "env": "ENV_NAME" }`
  - default mutation blocking for non-`GET` methods unless `metadata.allow_mutation=true`
- Failed tool-call records and user-safe fallback replies for blocked or failed OpenAPI tool executions.
- `pnpm smoke:tools`, a real local smoke test that configures an OpenAPI tool, calls a local HTTP fixture, verifies the AI answer, and checks the stored tool-call output.

### Changed

- Demo order/subscription tools remain available as deterministic fixtures.
- Tool execution now supports both built-in demo tools and configured OpenAPI tools before the knowledge retrieval fallback.
- Workspace package versions are bumped to `0.9.0`.

### Validation

- `pnpm format:check`
- `pnpm exec prisma validate`
- `pnpm db:generate`
- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`
- `pnpm smoke:memory -- --help`
- `pnpm smoke:chatwoot -- --help`
- `pnpm smoke:channels -- --help`
- `pnpm smoke:tools -- --help`
- Real memory-mode smoke
- Real channel webhook smoke with generic webhook and signed Slack callback coverage
- Real OpenAPI tool executor smoke

### Known Limits

- There is no automatic OpenAPI spec import yet; tools are configured through the admin API.
- There is no model planner or human approval flow for high-risk operations yet.
- Tool secrets are injected through environment variables, not encrypted per-tool secret storage.
- Non-`GET` tool calls are blocked by default and should remain disabled for production unless an operator has explicitly reviewed the endpoint.
- Slack outbound replies, email, and Telegram provider APIs remain future adapter work.
- Retrieval still uses deterministic keyword scoring; embedding/vector retrieval remains a later release.

## 中文

这个 minor 版本把业务工具基础升级为可本地验证的 OpenAPI-style HTTP 执行器。

### 新增

- Active 的项目级工具定义现在可以执行 OpenAPI-style business tool。
- 支持通过 `keywords` 和正则 `extract` 规则做 metadata 驱动的意图匹配。
- HTTP 执行安全控制：
  - 通过 `metadata.allowed_hosts` 限制最终请求 host
  - 通过 `metadata.timeout_ms` 控制请求超时
  - 通过 `metadata.max_response_bytes` 限制响应大小
  - 通过 `metadata.response_path` 选择响应对象
  - 通过 `metadata.answer_template` 渲染回复
  - 通过 `metadata.auth = { "type": "bearer_env", "env": "ENV_NAME" }` 注入 bearer auth
  - 非 `GET` 方法默认阻断，除非显式配置 `metadata.allow_mutation=true`
- OpenAPI 工具被阻断或执行失败时，会记录 failed tool-call，并给用户返回安全失败提示。
- 新增 `pnpm smoke:tools` 真实本地 smoke test：配置 OpenAPI 工具、调用本地 HTTP fixture、验证 AI 回复并检查已保存的 tool-call output。

### 变更

- Demo 订单/订阅工具仍作为确定性 fixture 保留。
- Tool execution 现在支持内置 demo tools 和已配置的 OpenAPI tools，并在知识库检索 fallback 前执行。
- Workspace package versions 升级到 `0.9.0`。

### 验证

- `pnpm format:check`
- `pnpm exec prisma validate`
- `pnpm db:generate`
- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`
- `pnpm smoke:memory -- --help`
- `pnpm smoke:chatwoot -- --help`
- `pnpm smoke:channels -- --help`
- `pnpm smoke:tools -- --help`
- 真实 memory-mode smoke
- 真实 channel webhook smoke，覆盖 generic webhook 和已签名 Slack callback
- 真实 OpenAPI tool executor smoke

### 已知限制

- 还没有自动 OpenAPI spec import；工具仍通过管理端 API 配置。
- 还没有模型规划器或高风险操作人工审批流。
- 工具密钥通过环境变量注入，尚未实现工具级加密密钥存储。
- 非 `GET` 工具调用默认阻断；生产环境建议保持禁用，除非运营方已经明确审查 endpoint。
- Slack 出站回复、Email 和 Telegram provider API 仍是后续 adapter 工作。
- Retrieval 仍使用确定性的 keyword scoring；embedding/vector retrieval 延后到后续版本。
