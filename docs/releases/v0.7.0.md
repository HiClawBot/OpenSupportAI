# OpenSupportAI v0.7.0

## English

This minor release adds the first production-oriented knowledge indexing pipeline.

### Added

- Stored knowledge document content and content hashes.
- Knowledge document reindex API:
  - `POST /v1/admin/projects/{project_id}/knowledge/documents/{document_id}/reindex`
  - creates a `knowledge.index` async job
  - moves the document back to `pending`
- Real `knowledge.index` worker handler:
  - marks the document `indexing`
  - rebuilds all chunks from stored content
  - marks the document `indexed` with chunk-count metadata
  - marks the document `failed` with an error when indexing cannot complete
- Admin console reindex buttons and source/chunk/error visibility for knowledge documents.
- Protocol and data model updates for knowledge document metadata.

### Changed

- Bumped workspace package versions to `0.7.0`.
- Worker service now depends on `@opensupportai/rag` and `@prisma/client`.
- Prisma seed now stores demo knowledge document content and content hash.

### Validation

- `pnpm format:check`
- `pnpm exec prisma validate`
- `pnpm db:generate`
- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`
- `pnpm smoke:memory -- --help`
- `pnpm smoke:chatwoot -- --help`
- `pnpm smoke:channels -- --help`
- Real memory-mode smoke
- Real channel webhook smoke

### Known Limits

- Retrieval still uses deterministic keyword scoring.
- The schema remains pgvector-ready, but embedding/vector retrieval is deferred to a later release.
- `webhook.retry` worker execution is still a placeholder result; scheduling already exists.

## 中文

这个 minor 版本加入第一条面向生产的知识库索引管线。

### 新增

- Knowledge document 持久化保存原文内容和 content hash。
- Knowledge document reindex API：
  - `POST /v1/admin/projects/{project_id}/knowledge/documents/{document_id}/reindex`
  - 创建 `knowledge.index` async job
  - 将文档状态切回 `pending`
- 真实 `knowledge.index` worker handler：
  - 将文档标记为 `indexing`
  - 基于已保存原文重建全部 chunks
  - 成功后标记为 `indexed` 并写入 chunk-count metadata
  - 失败后标记为 `failed` 并写入错误信息
- Admin Console 为 knowledge document 新增 reindex 按钮，并展示 source、chunk count 和 error。
- Protocol 和数据模型补充 knowledge document metadata。

### 变更

- Workspace package versions 升级到 `0.7.0`。
- Worker service 新增 `@opensupportai/rag` 和 `@prisma/client` 依赖。
- Prisma seed 会保存 demo knowledge document 原文和 content hash。

### 验证

- `pnpm format:check`
- `pnpm exec prisma validate`
- `pnpm db:generate`
- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`
- `pnpm smoke:memory -- --help`
- `pnpm smoke:chatwoot -- --help`
- `pnpm smoke:channels -- --help`
- 真实 memory-mode smoke
- 真实 channel webhook smoke

### 已知限制

- Retrieval 仍使用确定性的 keyword scoring。
- Schema 继续保持 pgvector-ready，但 embedding/vector retrieval 延后到后续版本。
- `webhook.retry` worker 执行仍是 placeholder result；调度能力已经存在。
