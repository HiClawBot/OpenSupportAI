# OpenSupportAI v0.2.0

## English

This release adds the first production-foundation layer on top of the v0.1 runtime.

Highlights:

- Added project-scoped admin API key management:
  - `GET /v1/admin/projects/{project_id}/api-keys`
  - `POST /v1/admin/projects/{project_id}/api-keys`
  - `DELETE /v1/admin/projects/{project_id}/api-keys/{key_id}`
- Added API key identity tracking and `lastUsedAt` updates.
- Restricted project-scoped API keys to their own project and kept project creation root-admin only.
- Added Prisma `AuditLog` storage and admin audit-log API.
- Added ops health API: `GET /v1/admin/projects/{project_id}/ops/health`.
- Added webhook event management:
  - `GET /v1/admin/projects/{project_id}/webhooks/events`
  - `POST /v1/admin/projects/{project_id}/webhooks/events/{event_id}/retry`
- Webhook retries now schedule `webhook.retry` async jobs for worker-side processing.
- Added tests for API key lifecycle, audit logs, ops health, and webhook retry scheduling.

Validation:

- `pnpm format:check`
- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`
- `pnpm exec prisma validate`
- `pnpm db:generate`
- `pnpm smoke:memory -- --help`
- `pnpm smoke:chatwoot -- --help`

## 中文

这个版本在 v0.1 运行时之上补上第一层生产基础能力。

主要更新：

- 新增项目级 admin API key 管理：
  - `GET /v1/admin/projects/{project_id}/api-keys`
  - `POST /v1/admin/projects/{project_id}/api-keys`
  - `DELETE /v1/admin/projects/{project_id}/api-keys/{key_id}`
- 新增 API key 身份识别和 `lastUsedAt` 更新。
- 项目级 API key 只能访问自己的项目，创建新项目仍仅允许 root admin token。
- 新增 Prisma `AuditLog` 存储和管理端审计日志 API。
- 新增 ops health API：`GET /v1/admin/projects/{project_id}/ops/health`。
- 新增 webhook event 管理：
  - `GET /v1/admin/projects/{project_id}/webhooks/events`
  - `POST /v1/admin/projects/{project_id}/webhooks/events/{event_id}/retry`
- Webhook retry 会调度 `webhook.retry` async job，供后续 worker 处理器消费。
- 新增 API key 生命周期、审计日志、ops health 和 webhook retry 调度测试。

验证项：

- `pnpm format:check`
- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`
- `pnpm exec prisma validate`
