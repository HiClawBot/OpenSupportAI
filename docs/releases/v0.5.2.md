# OpenSupportAI v0.5.2

## English

This patch release turns more of the production foundation APIs into visible operator workflows in the admin console.

### Added

- Admin console Operations area for:
  - ops health
  - channel adapter catalog and diagnostics
  - generic webhook channel configuration
  - admin API keys
  - audit logs
  - async jobs
  - webhook events and retry scheduling
  - tool-call logs
- Admin actions to test channel adapters, create/revoke project admin API keys, queue async jobs, and schedule webhook event retries.
- Docker Compose validation notes for memory and channel smoke tests.

### Changed

- Upgraded GitHub Actions workflow actions to the current v6 release line.
- Updated README, release checklist, and Docker Compose docs to describe the operator UI and production secret requirements.
- Bumped workspace package versions to `0.5.2`.

### Validation

- `pnpm format:check`
- `pnpm exec prisma validate`
- `pnpm db:generate`
- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`
- `pnpm smoke:memory -- --help`
- `pnpm smoke:chatwoot -- --help`
- `pnpm smoke:channels -- --help`
- Real memory-mode smoke
- Real channel webhook smoke
- Browser verification of the admin console Operations area
