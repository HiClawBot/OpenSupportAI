# OpenSupportAI v0.8.0

## English

This minor release adds the first real provider channel: a locally testable Slack inbound adapter for Slack Events API callbacks.

### Added

- Slack channel adapter with `available` catalog status.
- Slack signing secret configuration through admin API and the admin console Operations area.
- Slack adapter test path that reports missing configuration and validates configured inbound readiness.
- Slack webhook endpoint:
  - `POST /v1/channel-webhooks/slack?public_key=pk_demo`
  - verifies Slack timestamp/signature headers
  - handles Slack URL verification challenges
  - normalizes Slack `event_callback` message events into OpenSupportAI conversations
  - stores webhook events and keeps repeated Slack `event_id` deliveries idempotent
- Channel smoke coverage for signed Slack callbacks, invalid signatures, duplicate events, admin channel visibility, and webhook event visibility.

### Changed

- Slack moved from a contract stub to a signed inbound adapter.
- Admin console channel operations now include Slack configuration alongside generic webhook configuration.
- `@opensupportai/adapter-channels` now uses Node crypto for provider signature verification.
- Workspace package versions are bumped to `0.8.0`.

### Validation

- `pnpm format:check`
- `pnpm exec prisma validate`
- `pnpm db:generate`
- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`
- `pnpm smoke:memory -- --help`
- `pnpm smoke:chatwoot -- --help`
- `pnpm smoke:channels -- --help`
- Real memory-mode smoke
- Real channel webhook smoke with generic webhook and signed Slack callback coverage

### Known Limits

- Slack outbound replies are not implemented yet.
- Email and Telegram remain contract stubs.
- Slack signature verification is covered for JSON callback payloads in the current Fastify path; raw-body capture should be hardened before high-volume public Slack deployments behind arbitrary proxies.
- Retrieval still uses deterministic keyword scoring; embedding/vector retrieval remains a later release.
- `webhook.retry` worker execution is still a placeholder result; scheduling already exists.

## 中文

这个 minor 版本新增第一条真实 provider channel：可本地验证的 Slack 入站 adapter，面向 Slack Events API callback。

### 新增

- Slack channel adapter 在 catalog 中进入 `available` 状态。
- 管理端 API 和 Admin Console Operations 区域支持配置 Slack signing secret。
- Slack adapter test 路径可报告缺失配置，并验证已配置的入站 readiness。
- Slack webhook endpoint：
  - `POST /v1/channel-webhooks/slack?public_key=pk_demo`
  - 校验 Slack timestamp/signature headers
  - 处理 Slack URL verification challenge
  - 将 Slack `event_callback` message event 归一化为 OpenSupportAI 会话
  - 存储 webhook event，并对重复 Slack `event_id` 保持幂等
- Channel smoke 覆盖已签名 Slack callback、非法签名、重复事件、管理端 channel 可见性和 webhook event 可见性。

### 变更

- Slack 从契约 stub 升级为签名校验后的入站 adapter。
- Admin Console channel operations 现在包含 Slack 配置，与 generic webhook 配置并列。
- `@opensupportai/adapter-channels` 现在使用 Node crypto 做 provider signature verification。
- Workspace package versions 升级到 `0.8.0`。

### 验证

- `pnpm format:check`
- `pnpm exec prisma validate`
- `pnpm db:generate`
- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`
- `pnpm smoke:memory -- --help`
- `pnpm smoke:chatwoot -- --help`
- `pnpm smoke:channels -- --help`
- 真实 memory-mode smoke
- 真实 channel webhook smoke，覆盖 generic webhook 和已签名 Slack callback

### 已知限制

- Slack 出站回复尚未实现。
- Email 和 Telegram 仍是契约 stub。
- 当前 Fastify 路径已覆盖 JSON callback payload 的 Slack signature verification；在任意代理后的高流量公开 Slack 部署前，还应继续强化 raw-body capture。
- Retrieval 仍使用确定性的 keyword scoring；embedding/vector retrieval 延后到后续版本。
- `webhook.retry` worker 执行仍是 placeholder result；调度能力已经存在。
